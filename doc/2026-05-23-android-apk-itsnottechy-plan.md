# Android APK for Onsective (itsnottechy.cloud) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a signed, installable Android APK from `apps/mobile/` that talks to the live backend at `https://api.itsnottechy.cloud`.

**Architecture:** Two-file config change (apiUrl + deep-link host), then Expo prebuild to generate a native Android Gradle project, then a one-time keystore generation, then `./gradlew assembleRelease`. No application code is modified — all downstream code reads the API URL via `Constants.expoConfig.extra.apiUrl`.

**Tech Stack:** Expo SDK 51, React Native 0.74, Gradle, Android SDK (already installed at `~/Library/Android/sdk`), JDK (already at `/usr/bin/java`), pnpm workspaces, Turborepo.

**Note on TDD:** This plan does not follow the "write failing test first" pattern because every task is either a config edit, a one-time keystore generation, or a build invocation — there is no application logic to test. Verification is via curl, file inspection (`unzip -p`), and a successful Gradle build. If you find yourself wanting a unit test for any step here, stop and reconsider.

## Task 1: Pre-flight — verify backend is reachable

**Files:** none modified.

**Why first:** If the VPS backend is down or HTTPS is misconfigured, the APK will build fine but fail at first launch. Catch it now, not after a 20-minute Gradle build.

- [ ] **Step 1: Hit the API over HTTPS**

Run:
```bash
curl -sS -o /dev/null -w "HTTP %{http_code}\nTLS %{ssl_verify_result}\nTotal %{time_total}s\n" https://api.itsnottechy.cloud/health
```

Expected: `HTTP 200` (or `HTTP 404` if there's no `/health` route — that's still proof the server is responding over TLS). If `HTTP 000`, DNS/TLS/firewall is broken — STOP and report to the user.

- [ ] **Step 2: If `/health` returned 404, try the Swagger doc instead**

Run:
```bash
curl -sS -o /dev/null -w "HTTP %{http_code}\n" https://api.itsnottechy.cloud/docs
```

Expected: `HTTP 200` (NestJS Swagger UI per `services/api`'s `/docs` mount).

- [ ] **Step 3: Report findings to user before proceeding**

If any check failed, surface the failure plainly with the exact curl output. Do not start editing files until the user confirms how to proceed.

## Task 6: Generate release keystore

**Files:** creates `apps/mobile/android/app/onsective-release.keystore`.

- [ ] **Step 1: Generate the keystore**

Run (from repo root):
```bash
keytool -genkeypair -v \
  -keystore apps/mobile/android/app/onsective-release.keystore \
  -alias onsective \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass onsective2026 -keypass onsective2026 \
  -dname "CN=Onsective, OU=Mobile, O=Itsnottechy, L=Unknown, ST=Unknown, C=IN"
```

Expected: prints `[Storing apps/mobile/android/app/onsective-release.keystore]` and exits 0.

**Note on passwords:** Using `onsective2026` here for unattended generation. We will note this clearly in the final report so the user can rotate them via `keytool -storepasswd` before any production use.

- [ ] **Step 2: Verify the keystore loaded**

Run:
```bash
keytool -list -v -keystore apps/mobile/android/app/onsective-release.keystore -storepass onsective2026 | head -25
```

Expected: shows `Alias name: onsective`, `Owner: CN=Onsective...`, and a SHA-256 fingerprint. **Capture that SHA-256 fingerprint** — it's needed later for `assetlinks.json` (out of scope for this plan but the user will need it).

## Task 8: Build the release APK

**Files:** generates `apps/mobile/android/app/build/outputs/apk/release/app-release.apk`.

- [ ] **Step 1: Build**

Run (from repo root):
```bash
cd apps/mobile/android && ./gradlew assembleRelease
```

Expected: ends with `BUILD SUCCESSFUL in Xm Ys`. First build takes 10–20 minutes (downloads Android Gradle Plugin, NDK pieces, dependencies). Subsequent builds take 1–3 minutes.

If the build fails:
- Read the actual error (Gradle prints a tall stack trace; the real error is usually the last `> Task :` line that failed).
- **Common cause:** JDK version. Modern Expo/RN needs JDK 17. Check with `java -version`; if not 17, install via `brew install --cask temurin@17` and re-run with `export JAVA_HOME=$(/usr/libexec/java_home -v 17)`.
- **Common cause:** Missing Android SDK platform. Open Android Studio → SDK Manager → install API level matching `compileSdkVersion` in `apps/mobile/android/build.gradle`.

- [ ] **Step 2: Verify the APK exists**

Run:
```bash
ls -lh apps/mobile/android/app/build/outputs/apk/release/app-release.apk
```

Expected: file exists, size between roughly 30 MB and 80 MB.

- [ ] **Step 3: Verify the APK contains the right package and apiUrl**

Run:
```bash
unzip -p apps/mobile/android/app/build/outputs/apk/release/app-release.apk AndroidManifest.xml | strings | grep -E "com.onsective|itsnottechy" | head -5
```

Expected: see `com.onsective.app` and `shop.itsnottechy.cloud` (the apiUrl is in compiled JS bundle, not the manifest — that's verified at runtime, not here).

- [ ] **Step 4: (Optional) Verify the JS bundle has the new apiUrl**

Run:
